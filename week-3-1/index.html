Ajax

Ajax is a technique that allows web pages to communicate asynchronously with a server and dynamically update web pages without reloading. Allows data to be sent and received at the background. Web pages will update in response to user events while the rest of the program continue to run. Ajax stands for asynchronous JavaScript XML. Asynchronous meaning the program does wait for response to. It continues to run waiting for an event to fire when response is received. JavaScript send and receive data allowing updating of content in real-time.

Same -origin policy

Browsers have same origin policy which blocks all requests from a domain that is different from the page making the request. The purpose is to stop any malicious JavaScript being run from an external source. Cross-origin resource sharing (CORS) provides a solution to this problem.

APIS

Application programming interface is a collection of methods that allow external access to another program or service. We have to enable CORS to access information from an API and sometimes authentication is required.

The Fetch API

  • a living standard for requesting and sending data asynchronously across a network.
  • uses promises to avoid callback hell, and also streamlines a number of concepts that had become cumbersome when using the XMLHttpRequest object
