JavaScript Functions

  1. javascript functions are first-class objects because they can be passed around like any other values.
  2. They have properties and methods
  3. They can be passed as parameters to other functions and can be returned by other functions.
  4. They are a flexible tool to work with.
Functions have a length property that returns the number of parameters the function has. They also have call() and apply() methods. These two works the same way, the difference is in the way the arguments are passed. For apply() the arguments are passed as an array regardless of the number of arguments of the function. They can be used to set the value of this inside a function to an object that is provided as the first argument.

I can create custom properties for functions. For example, adding the description property that explains what the function does.

Memoization

A useful feature when working with functions that take some time to compute. The result can be saved in a catch property to be returned whenever the same argument is used again. Example of storing a result in a catch below.

function square(x){
square.cache = square.cache || {};
square.cache[x] = x*x;
} return square.cache[x]}

Recursive Functions

- functions that invoke until a certain condition is met.
- Useful when there is some iteration involved.

Callbacks types

- These are functions passed to other functions as arguments and then invoked inside the function they are passed to.
- They are used to facilitate event-driven asynchronous programming.
- Allows the program to run than wait for certain events to occur.
- Callback functions returns other functions.

Async Functions

- functions are preceded by the async keyword and allow you to write asynchronous code as if it was synchronous.
- achieved by using the await operator before an asynchronous function.
- wrap the return value of the function in a promise that can then be assigned to a variable. The next line of code is not executed until the promise is resolved.
