Templating

Templates make it easy to create reusable code. Templating is big on the web and most sites do server-side templating. This is achieved by using web components or application frameworks. It provides an easy way of defining reusable fragments of HTML that can be manipulated the same you would the contents of a document itself.

The contents of a template get parsed like regular HTML. The differences between the two are:

  1. Contents of a template are not rendered
  2. Scripts inside it won’t run
  3. Tags will not be evaluated
  4. It will not use external resources
Content is a powerful property. DocumentFragment instances provide an API for manipulating It (content) like a global document. Inserting DocumentFragment into the DOM is faster than manipulating elements’ innerHTML property because the DOM structure already exist. We use templates to into the DOM. We make use of the clone method and set it to true. Templates are widely supported.

Single Page Applications enable users to get content in response to navigation without making a request to the server for new HTML. They use the window. Location method and rely on a router.
